# MyPersonalShelf

MyPersonalShelf는 로컬 파일, 폴더, 문서, 영상, 음악, 이미지, 메모, 웹 링크를 한곳에 모아두는 로컬 우선 데스크톱 앱입니다. 단순 북마크 앱보다 넓은 개념의 개인 콘텐츠 선반을 목표로 합니다.

영어 문서는 [README.md](README.md)를 참고하세요.

## 프로젝트 방향

- 자주 바뀌는 과제나 업무 기록보다 오래 보관할 자료를 중심으로 다룹니다.
- 예: 강의 자료, 소설 txt/md 파일, 저장한 영상, 음악 폴더, 참고 링크, 개인 아카이브.
- 파일을 앱 내부로 복사하기보다 경로와 메타데이터를 먼저 저장합니다.
- 대시보드는 내가 자주 보는 자료를 카드처럼 배치하는 개인 선반 느낌을 지향합니다.
- 실제로 사용하면서 기능과 UI를 점진적으로 개선하는 프로젝트입니다.

## 현재 기능

- 고정 카드, 최근 열린 항목, 자주 여는 항목이 있는 대시보드.
- 로컬 파일, 로컬 폴더, 웹 링크, 직접 작성 메모 등록.
- 타입, 태그, 컬렉션, 색상, 고정 여부, 대시보드 배치 설정.
- 라이브러리에서 항목을 선택하면 오른쪽 미리보기로 내용 확인.
- 데스크톱 앱처럼 동작하는 기본 조작 방식:
  - 한 번 클릭: 선택하고 미리보기.
  - 더블클릭: 항목 열기.
  - `Ctrl+Enter`: 선택된 항목 열기.
- `txt`, `md`, `markdown` 등 텍스트 문서 읽기.
- Markdown 문서를 원문 그대로가 아니라 읽기 좋은 형식으로 렌더링.
- 설정에 따라 문서, 영상, 음악, 이미지를 별도 뷰어 창으로 열기.
- Tauri asset URL을 통한 로컬 영상/음악/이미지 재생 및 미리보기.
- 문서 읽기 위치와 영상/음악 이어보기 위치 저장.
- 저장한 웹 링크를 기본 브라우저로 열기.
- 저장한 폴더를 시스템 파일 탐색기로 열기.
- 언어, 배경색, 표면색, 글자색, 강조색, 읽기 폭, 줄간격, 글자 크기, 읽기 창 방식 커스터마이징.
- 일반 검색과 `tag:읽기`, `type:video`, `collection:Media`, `open:novel` 같은 명령형 검색.

## 데이터 저장 방식

- 데스크톱 앱에서는 Tauri를 통해 SQLite에 앱 상태를 저장합니다.
- 브라우저 미리보기 환경에서는 localStorage를 fallback으로 사용합니다.
- 로컬 파일은 기본적으로 앱 내부에 복사하지 않고 경로와 메타데이터를 저장합니다.
- 읽기 진행률과 미디어 이어보기 위치는 별도 저장되어 다음 실행 때 복원됩니다.

## 기술 스택

- Desktop: Tauri
- Frontend: React + TypeScript
- Build: Vite
- Local data: SQLite, browser preview에서는 localStorage fallback
- Styling: 일반 CSS
- Layout: 반응형 카드 그리드와 대시보드 활동 패널

## 앞으로 넣어볼 기능

- 웹 링크 미리보기 카드, favicon, 페이지 제목 자동 저장.
- YouTube / YouTube Music 링크 전용 처리.
- 브라우저 북마크 가져오기.
- EPUB/PDF 뷰어.
- 영상 자막 지원.
- AI 요약과 태그 추천.
- 백업/내보내기/복원 흐름.
- 플러그인식 확장 구조.

## 개발 방법

전체 Tauri 데스크톱 앱을 실행하려면 Rust/Cargo가 필요합니다.

```bash
npm install
npm run dev
npm run tauri dev
```

앱 표시 이름은 `src/lib/appConfig.ts`에 분리되어 있어 나중에 프로젝트 이름을 바꿀 때 내부 패키지명과 별도로 조정할 수 있습니다.
